# 数据展示视觉优化设计

## 背景

当前项目登录后的组织管理者与志愿者页面，已经具备较完整的数据展示能力，但列表页的视觉语言仍然偏分散：

- 部分页面以表格为主，部分页面以卡片和工具区组合为主，信息密度和节奏不一致。
- 筛选、搜索、操作、分页的布局方式在不同页面中差异较大，缺少统一的数据工作台体验。
- 状态标签、操作按钮、表格容器的表达存在一定程度的视觉竞争，削弱了主数据本身的可读性。

用户提供的参考图明确表达了目标方向：不是整页换肤，也不是统一成纯白页面，而是将“数据展示形式”优化为更简约、大气、规整的企业数据台风格。

## 目标

本次优化聚焦登录后页面中的数据展示层，统一表格型页面的视觉结构与交互节奏，使其更接近参考图的表达方式，同时保持现有角色区分、背景氛围与业务逻辑不变。

具体目标：

- 统一登录后列表型数据页的视觉语法。
- 将表格、筛选条、操作区、分页区、状态标签收敛为更规整的企业后台风格。
- 优先通过共享组件升级覆盖组织管理者和志愿者页面。
- 尽量不改动页面业务逻辑、路由结构、接口调用和非数据展示模块。
- 不影响登录前页面与其他通用组件的现有使用。

## 非目标

以下内容不在本次范围内：

- 登录前页面视觉调整。
- Dashboard、统计概览、趋势图等非列表型数据模块的大改。
- 将所有页面统一改成白底或完全复制参考图配色。
- 接口、筛选逻辑、分页逻辑、抽屉逻辑、导出逻辑等业务行为变更。
- 大面积重写页面模板或推翻现有角色主题设计。

## 设计原则

### 1. 数据优先

用户进入页面后，应首先看到数据本身和操作主线，而不是被重色块、重阴影或装饰性元素分散注意力。

### 2. 形式统一，页面个性保留

统一的是“数据展示方式”，不是整个页面皮肤。组织管理者与志愿者页面可以继续保留自身主题色和背景，但数据区域要有一致的结构、节奏和可读性。

### 3. 共享层优先

优先在共享数据组件层完成改造，让多个页面自动继承样式升级，减少逐页定制，降低回归风险。

### 4. 低风险演进

在不改变组件使用方式和业务逻辑的前提下，通过样式、容器结构和轻量变体能力完成优化。

## 组件改造策略

### 共享组件

本次改造优先落在以下共享层：

#### `src/components/data-list/DataTable.vue`

负责统一表格主体的展示风格：

- 统一表头层级、字重、文字颜色与弱背景处理。
- 统一行高、单元格留白、分隔线样式。
- 优化行 hover、选中态和操作列视觉关系。
- 统一空态、加载态与表格主体的融合度。

#### `src/components/data-list/DataToolbar.vue`

负责统一搜索、筛选、动作按钮和分页设置这类上层操作区：

- 强化左右分区和工具带结构。
- 统一控件间距、对齐方式和换行逻辑。
- 让筛选项与动作按钮的优先级更接近参考图的工作台体验。

#### `src/components/data-list/DataListPage.vue`

负责统一数据页骨架：

- 规范 header、toolbar、body 之间的垂直节奏。
- 统一数据容器的留白和区块边界。
- 为数据页提供更稳的承载结构，但不强制覆盖页面原始背景风格。

#### `src/components/data-list/StatusBadge.vue`

负责统一状态标签风格：

- 保留状态色语义。
- 降低饱和度、体积感和视觉攻击性。
- 让标签成为辅助信息，而不是主视觉焦点。

### 页面级适配

仅在共享层不足以覆盖时，才对个别目标页做轻量模板调整，例如：

- 调整筛选控件顺序。
- 统一列表上方统计文案的位置和优先级。
- 收敛分页信息与操作按钮的排列方式。

页面级调整必须满足两个约束：

- 不改变原有业务流程。
- 不重写页面主体结构。

## 视觉规范

### 表格

目标效果接近参考图中的企业数据列表：

- 表头明确但不厚重。
- 行与行之间通过轻分隔线和适度留白建立秩序感。
- 文本信息优先，装饰性弱化。
- 操作列简洁清晰，避免过度按钮化。

### 工具区

工具区采用“先筛选，再查看，再操作”的信息顺序：

- 搜索、筛选位于更靠前位置。
- 导出、批量操作等动作位于操作侧。
- 每页条数、分页控制等辅助能力降低视觉优先级。

### 状态标签

状态标签遵循“可识别、不过载”的原则：

- 用低饱和底色与中等对比文字区分状态。
- 保持统一圆角、字号和内边距。
- 不让标签颜色盖过主数据文本。

### 容器与留白

数据区域强调规整的内容边界：

- 使用更统一的容器圆角、边框和内边距。
- 在有主题背景的页面上，让数据区显得清爽但不脱离整体风格。
- 避免把页面改成完全一致的白底后台。

### 分页与辅助信息

分页与统计文案为辅助层：

- 保留信息完整性。
- 降低字号、对比和视觉重量。
- 避免与表格标题、筛选操作区抢层级。

## 兼容性与风险控制

为了避免影响其他组件使用，本次实现遵循以下边界：

- 尽量通过数据展示组件自身样式命名空间控制改动。
- 不修改通用按钮、输入框、日期选择器的底层交互行为。
- 若共享组件需要新增视觉变体，优先通过可选 prop 或局部 class 控制，而不是直接覆盖全站样式。
- 不依赖页面外层背景颜色来完成核心展示效果。

主要风险与对应策略：

- 风险：共享组件改动影响非目标页面。
  策略：优先限制在数据展示组件命名空间内，必要时增加变体开关。

- 风险：个别页面结构差异较大，无法完全自动继承。
  策略：仅对目标页增加轻量布局补位，不推翻原模板。

- 风险：不同角色页面风格过于趋同，丢失识别度。
  策略：只统一数据层语法，不统一整页背景和头部设计。

## 验证方案

实现完成后至少验证以下内容：

1. 组织管理者与志愿者侧目标列表页都能正常渲染。
2. 搜索、筛选、分页、导出、行点击、抽屉等既有交互不受影响。
3. 登录前页面与非目标页面没有明显视觉回归。
4. `DataTable`、`DataToolbar`、`DataListPage`、`StatusBadge` 的修改在目标页上表现一致。
5. `Volunteer/Activities.vue` 等非纯表格页只发生轻量收敛，不被错误表格化。

## 实施建议

建议分两步执行：

### 第一步：共享层升级

- 完成 `DataTable`、`DataToolbar`、`DataListPage`、`StatusBadge` 的统一样式升级。
- 在全局样式文件中补充最小必要的数据展示样式规则。

### 第二步：页面适配与验证

- 逐页检查目标页面的工具区、列表区和分页区。
- 对少数不完全适配的页面做轻量模板调整。
- 完成回归验证，确保未影响其他登录后页面和登录前页面。

## 决策摘要

本次方案选择：

- 仅优化登录后页面。
- 仅聚焦数据展示形式，不整页换肤。
- 优先通过共享组件升级覆盖多页面。
- 列表型页面全面纳入，Dashboard 等非列表型页面不作为本次重点。
- 保留现有角色主题和页面外层气质，只让数据层更接近参考图的企业数据台表达。
